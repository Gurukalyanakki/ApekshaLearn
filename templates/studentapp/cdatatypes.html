Data Types in C

Each variable in C has an associated data type that specifies the type of data it can store, like integer, character, floating, double, etc. Each data type requires different amounts of memory and has specific operations that can be performed on it.

Classification of Data Types in C

  • Primitive Data Types: Basic data types used for representing simple values like integers, float, characters, etc.
    Examples: int, char, float, double, void
  • Derived Types: Derived from primitive data types.
    Examples: array, pointers, function
  • User-Defined Data Types: Defined by the user to suit specific requirements.
    Examples: structure, union, enum

Understanding C's data types is critical for writing efficient programs. If you're interested in learning more about data types and their interaction with data structures, consider exploring an online course in C programming with data structures.

Main Primitive Data Types in C

Data Type Description Range Size Format Specifier
int Used to store integer numbers (positive, negative, or zero without a decimal part). -2,147,483,648 to 2,147,483,647 4 bytes %d
unsigned int Stores positive integers only, without negative values. 0 to 4,294,967,295 4 bytes %u
short int Smaller than int by 2 bytes, storing smaller range values. -32,768 to 32,767 2 bytes %hd
long int Larger version of int, used for storing larger range values. -263 to 263 - 1 8 bytes (on most systems) %ld
float Stores decimal values, with single-precision floating point. ±3.4E-38 to ±3.4E+38 4 bytes %f
double Double-precision floating point, stores larger decimal values than float. ±1.7E-308 to ±1.7E+308 8 bytes %lf
char Used to store single characters. -128 to 127 1 byte %c

Syntax of Integer Declaration

We use the int keyword to declare an integer variable:

int var_name;

Variations of Integer

  • unsigned int: Stores only positive values.
  • short int: Smaller in size than int (2 bytes).
  • long int: Larger in size than int.
  • unsigned short int: Similar to unsigned int but works with short int.

Note: The size of an integer data type may vary depending on the compiler. Use the sizeof operator to check the actual size of any data type on your system.

Sample Code


#include <stdio.h>


    int main() {
        // Fundamental Data Types
        int intVar = 10;              // Integer: typically 4 bytes
        float floatVar = 5.75;        // Floating Point: typically 4 bytes
        double doubleVar = 19.99;     // Double Precision: typically 8 bytes
        char charVar = 'A';           // Character: typically 1 byte
    
        // Displaying the sizes of each data type
        printf("Size of int: %zu bytes\n", sizeof(intVar));
        printf("Size of float: %zu bytes\n", sizeof(floatVar));
        printf("Size of double: %zu bytes\n", sizeof(doubleVar));
        printf("Size of char: %zu byte\n", sizeof(charVar));
    
        // Derived Data Types
        int array[5] = {1, 2, 3, 4, 5}; // Array of integers
        int *pointerVar = &intVar;      // Pointer to integer
        struct {                         // Structure example
            int id;
            char name[20];
        } student = {1, "John Doe"};
    
        // Displaying values
        printf("Integer: %d\n", intVar);
        printf("Float: %.2f\n", floatVar);
        printf("Double: %.2f\n", doubleVar);
        printf("Char: %c\n", charVar);
        printf("Array: ");
        for(int i = 0; i < 5; i++) {
            printf("%d ", array[i]);
        }
        printf("\nPointer: %d\n", *pointerVar);
        printf("Struct: ID: %d, Name: %s\n", student.id, student.name);
    
        return 0;
    }

The bool in C is a fundamental data type that can hold one of two values: true or false. It is used to represent logical values and is commonly used in programming to control the flow of execution in decision-making statements such as if-else statements, while loops, and for loops. In this article, we will explore how to use the bool data type in C.

Boolean in C

In C, the bool data type is not a built-in data type. However, the C99 standard for C language supports bool variables. Boolean can store values as true-false, 0-1, or can be yes-no. It can be implemented in C using different methods as mentioned below:

1. Using Header File “stdbool.h”

To use bool in C, you must include the header file “stdbool.h”. After including the stdbool.h library, we can use the data type as boolean is not available with stdio.h library.

Sample Code


#include <stdbool.h>
#include <stdio.h>

// Main Function
int main() {
    // Boolean data types declared
    bool a = true;
    bool b = false;

    printf("True : %d\n", a);
    printf("False : %d", b);

    return 0;
}

Output

True : 1
False : 0

2. Using the Enumeration Type

Alternatively, you can implement bool in C using an enumeration type. Here rather than importing the library, we declare an enumeration type to use bool as the data type.

Sample Code


#include <stdio.h>

typedef enum { false, true } bool;

int main() {
    bool a = true;
    bool b = false;

    printf("True : %d\n", a);
    printf("False : %d", b);

    return 0;
}

Output

True : 1
False : 0

3. Using Define to Declare Boolean Values

In this case, the false value is assigned the integer value of 0, and the true value is assigned the integer value of 1.

Sample Code


#define bool int
#define false 0
#define true 1

int main() {
    bool a = true;
    bool b = false;

    printf("True : %d\n", a);
    printf("False : %d", b);

    return 0;
}

Output

True : 1
False : 0

Type Conversion in C

Last Updated: 11 Oct, 2024

Type conversion in C is the process of converting one data type to another. The type conversion is only performed to those data types where conversion is possible. Type conversion is performed by a compiler. In type conversion, the destination data type can’t be smaller than the source data type. Type conversion is done at compile time and it is also called widening conversion because the destination data type can’t be smaller than the source data type. There are two types of conversion:

1. Implicit Type Conversion

Implicit Type Conversion, also known as ‘automatic type conversion’:

  • A. Done by the compiler on its own, without any external trigger from the user.
  • B. Generally takes place when in an expression more than one data type is present.
  • C. All data types of the variables are upgraded to the data type of the variable with the largest data type.
  • D. It is possible for implicit conversions to lose information, signs can be lost, and overflow can occur.

Example of Implicit Conversion

Sample Code


    #include <stdio.h>
    int main() {
        int x = 10; // integer x
        char y = 'a'; // character y
    
        // y implicitly converted to int. ASCII value of 'a' is 97
        x = x + y;
    
        // x is implicitly converted to float
        float z = x + 1.0;
    
        printf("x = %d, z = %f", x, z);
        return 0;
    }

Output

x = 107, z = 108.000000

2. Explicit Type Conversion

This process is also called type casting and it is user-defined. Here the user can typecast the result to make it of a particular data type.

Example of Explicit Type Conversion

Sample Code


    #include <stdio.h>
    
    int main() {
        double x = 1.2;
    
        // Explicit conversion from double to int
        int sum = (int)x + 1;
    
        printf("sum = %d", sum);
    
        return 0;
    }

Output

sum = 2

Another Example of Explicit Type Conversion

Sample Code


    #include <stdio.h>
    
    int main() {
        float a = 1.5;
        int b = (int)a;
    
        printf("a = %f\n", a);
        printf("b = %d\n", b);
    
        return 0;
    }

Output

a = 1.500000
b = 1

Advantages of Type Conversion

Type Safety: Type conversions help ensure that data is stored and processed in the correct data type, avoiding potential type mismatches and errors.

Improved Readability: Explicitly converting data between types clarifies the code's intent, making it easier to understand.

Enhanced Performance: Type conversions can optimize performance by converting data to more efficient data types for processing.

Compatibility: Conversions allow data to be handled between types that are not directly compatible, facilitating interactions with various APIs and libraries. .

Disadvantages of Type Conversion

Loss of Precision: Converting from a larger to a smaller data type may truncate some data, leading to loss of information.

Overflow and Underflow Risks: When converting data types, especially from smaller to larger types, there's a risk of overflow (too large) or underflow (too small) if values exceed the range of the target type.

Unexpected Behavior: Conversions between signed and unsigned types or between floating-point and integer types can yield results that are counterintuitive, causing logical errors.

Confusing Syntax: The syntax for type conversions can be complex and may make the code harder to read and maintain..

Conclusion

Type conversion is essential in C programming, allowing for flexibility and control over how data types are handled. Understanding both implicit and explicit conversions can help you write better, more efficient code.
