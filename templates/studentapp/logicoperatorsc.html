Conditional (Ternary) Operator in C

The conditional operator, also known as the ternary operator, is a concise alternative to the if-else statement in C.

This operator allows decision-making with a compact syntax, making code simpler when only a single action depends on a condition.

Syntax of Conditional/Ternary Operator in C:

Basic Syntax:

variable = Expression1 ? Expression2 : Expression3;

Here, Expression1 is the condition to evaluate. If true, Expression2 is executed; otherwise, Expression3 is executed.

Alternate Syntaxes:

variable = (condition) ? Expression2 : Expression3;
(condition) ? (variable = Expression2) : (variable = Expression3);

Visualizing the Ternary Operator as an if-else Statement

The ternary operation:

variable = condition ? Expression2 : Expression3;

is equivalent to:

if (condition) {
variable = Expression2;
} else {
variable = Expression3;
}

Note: The ternary operator has a lower precedence than many other operators, so parentheses should be used carefully to avoid unexpected behavior.

Working of the Ternary Operator

The conditional operator evaluates the condition in three main steps:

  1. Expression1 (the condition) is evaluated.
  2. If true, Expression2 is executed.
  3. If false, Expression3 is executed.

The result is returned and assigned to the variable (if applicable).

Example Programs

Example 1: Find the Greater of Two Numbers Using the Ternary Operator

Sample Code

#include <stdio.h>

int main() {
int m = 5, n = 4;

(m > n) ? printf("m is greater than n: %d > %d", m, n)
        : printf("n is greater than m: %d > %d", n, m);

return 0;
}

Output:

m is greater than n: 5 > 4

Example 2: Check Leap Year Using the Ternary Operator

#include <stdio.h>

int main() {
int yr = 1900;

(yr % 4 == 0) ? (yr % 100 != 0 ? printf("The year %d is a leap year", yr)
                : (yr % 400 == 0 ? printf("The year %d is a leap year", yr)
                                 : printf("The year %d is not a leap year", yr)))
              : printf("The year %d is not a leap year", yr);

return 0;
}

Output:

Sample Code

The year 1900 is not a leap year

Conclusion

The conditional (ternary) operator in C provides a shorthand way to write conditional assignments and expressions.

It's ideal for simple conditions but can make code harder to read when used for complex logic.

Operator Precedence and Associativity in C

Operator precedence and associativity in C help determine which operators are prioritized in expressions. This avoids unnecessary parentheses and ambiguity in evaluation order.

Operator Precedence and Associativity Table

The table below lists operators from highest to lowest precedence, along with their associativity:

Precedence Operator Description Associativity
1() [] . -> ++ --Parentheses, array, dot, pointer, postfix increment/decrementLeft-to-Right
2++ -- + - ! ~ (type) * & sizeofPrefix increment/decrement, unary operators, cast, dereference, addressRight-to-Left
3* / %Multiplication, division, modulusLeft-to-Right
15,Comma (expression separator)Left-to-Right

Mnemonic for Remembering Precedence: PUMA’S REBL TAC

P = Postfix, U = Unary, M = Multiplicative, A = Additive, S = Shift, R = Relational, E = Equality, B = Bitwise, L = Logical, T = Ternary, A = Assignment, C = Comma.

Examples of Operator Precedence and Associativity

1. Example of Operator Precedence

Sample Code

// C Program to illustrate operator precedence
#include <stdio.h>

int main() {
    printf("10 + 20 * 30 = %d", 10 + 20 * 30);
    return 0;
}

Output: 10 + 20 * 30 = 610

2. Example of Operator Associativity

Sample Code

// C Program to illustrate operator associativity
#include <stdio.h>

int main() {
    printf("100 / 5 % 2 = %d", 100 / 5 % 2);
    return 0;
}

Output: 100 / 5 % 2 = 0

3. Precedence and Associativity in Complex Expressions

Sample Code

// C Program to illustrate precedence and associativity
#include <stdio.h>

int main() {
    int exp = 100 + 200 / 10 - 3 * 10;
    printf("100 + 200 / 10 - 3 * 10 = %d", exp);
    return 0;
}

Output: 100 + 200 / 10 – 3 * 10 = 90

Important Points

  1. Associativity is only used when there are two operators of the same precedence. It doesn’t define the order in which operands of a single operator are evaluated.
  2. We can use parentheses to change the evaluation order.
  3. All operators with the same precedence have the same associativity.
  4. Postfix ++ has higher precedence than prefix ++, with different associativity.
  5. The comma operator has the lowest precedence and should be used cautiously.
  6. There’s no chaining of comparison operators in C, unlike Python.

Example of No Chaining in Comparison Operators

Sample Code

// C Program to show lack of chaining in comparison operators
#include <stdio.h>

int main() {
    int a = 10, b = 20, c = 30;
    if (c > b > a)
        printf("TRUE");
    else
        printf("FALSE");
    return 0;
}

Output: FALSE

Conclusion

Knowing operator precedence and associativity helps write efficient code and understand code from other programmers without excessive use of parentheses.
